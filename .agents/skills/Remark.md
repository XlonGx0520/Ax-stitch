# Agent 技能包中文使用说明与最佳流程

## 技能包简介

本项目集成了 `google-labs-code/stitch-skills` 技能包，为日常的开发、设计和文档编写提供了强大的 Agent 能力支持。

### 1. stitch-design（核心设计中心）
- **作用**：Stitch 设计相关任务的统一入口。负责提示词优化、建立设计系统规范（`DESIGN.md`）以及生成/编辑高保真界面。
- **使用时机**：需要生成新页面视觉原型，或修改现有页面 UI 时。
- **调用方法**：直接在对话框中向 Agent 提出设计需求（如：“帮我设计一个深色模式的登录页”，Agent 会自动调用此技能），或显式指定 `@stitch-design 帮我设计...`。

### 2. enhance-prompt（提示词增强大师）
- **作用**：把笼统的界面想法转换为针对 Stitch 优化的“高级设计提示词”，注入专业术语、对齐系统规范并优化排版结构。
- **使用时机**：当原始需求比较模糊，或之前生成的界面不理想时，先用它润色需求。
- **调用方法**：向 Agent 发送指令：“帮我优化一下这个 UI 原型的提示词：[你的粗略需求]”。

### 3. design-md（设计规范提炼师）
- **作用**：分析当前的 Stitch 项目，提取颜色规则、排版、间距和组件规范，生成语义化的 `DESIGN.md` 文档。
- **使用时机**：项目主页面设计基本定型，需要固化视觉规范以保持统一时。
- **调用方法**：向 Agent 发送指令：“请基于当前项目页面提炼设计系统到 DESIGN.md 中”。

### 4. react-components（前端代码落地）
- **作用**：将 Stitch 的视觉产物转换为基于真实 Vite 和 React 的可复用前端组件代码，并进行深度验证。
- **使用时机**：视觉原型验收通过，需要进入开发阶段将其转变为前端组件即可运行的代码时。
- **调用方法**：向 Agent 发送指令：“帮我把目前原型的 [某组件/页面] 转换为 React 代码组件”。

### 5. shadcn-ui（组件库开发专家）
- **作用**：基于当前最流行的 `shadcn/ui` 提供组件向导，指导发现、安装和定制基础组件并确保遵循最佳实践。
- **使用时机**：前端代码落地或二次开发中需要集成复杂基础组件（如日历、表格）时。
- **调用方法**：向 Agent 发送指令：“我想集成一个复杂下拉菜单，请用 shadcn-ui 技能帮我实现最佳实践”。

### 6. stitch-loop（全自动建站流）
- **作用**：指导 Agent 开启自动化“接棒循环”工作流（自我设计 -> 审查问题 -> 自我修复），自动构建多页面或复杂站点。
- **使用时机**：希望 Agent 全面负责一个复杂功能模块的自主迭代和试错打磨时。
- **调用方法**：向 Agent 发送指令：“使用 stitch-loop 模式为我构建一个 [你的功能/站点名称]”。

### 7. remotion（产品视频生成）
- **作用**：利用 Remotion 框架为现有的原型生成包含过渡、字幕解说和相机缩放的高质量漫游演示视频。
- **使用时机**：原型完成，需要向客户或团队做流媒体汇报演示时。
- **调用方法**：向 Agent 发送指令：“利用 remotion 帮我为目前的原型生成一段演示漫游视频”。
